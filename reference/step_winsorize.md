# Step for Winsorization

`step_winsorize()` creates a *recipe step* that applies winsorization to
numeric variables. Winsorization replaces extreme values below or above
specified quantile thresholds with those threshold values.

This step is useful to reduce the influence of outliers without fully
trimming observations.

## Usage

``` r
step_winsorize(
  recipe,
  ...,
  role = NA,
  trained = FALSE,
  probs = c(0.05, 0.95),
  skip = FALSE,
  id = recipes::rand_id("winsorize")
)
```

## Arguments

- recipe:

  A `recipe` object. The step will be added to the sequence of
  operations for this recipe.

- ...:

  One or more selector functions to choose which variables to apply
  winsorization to. If empty, all numeric variables are selected.

- role:

  Not used by this step since no new variables are created. Defaults to
  `NA`.

- trained:

  A logical indicating whether the step has been trained (i.e.,
  thresholds have been calculated).

- probs:

  A numeric vector of length 2 indicating the lower and upper quantile
  probabilities used to determine winsorization limits. Default is
  `c(0.05, 0.95)`.

- skip:

  A logical. Should the step be skipped when the recipe is baked by
  [`recipes::bake()`](https://recipes.tidymodels.org/reference/bake.html)?
  Default is `FALSE`.

- id:

  A character string used to identify this step. Defaults to a random
  string generated by `recipes::rand_id("winsorize")`.

## Value

An updated `recipe` object with the `step_winsorize` added to the
sequence of steps.

## Details

The actual winsorization limits are computed during the `prep()` phase
using the quantiles of the training data. During the `bake()` phase, all
values below the lower quantile threshold are replaced with that
threshold, and values above the upper quantile threshold are similarly
adjusted.
