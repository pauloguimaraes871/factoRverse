# Fit a Bayesian Hierarchical Regression Model and Summarize Posterior Metrics

This function fits a Bayesian hierarchical regression model using the
`brms` package and computes posterior summary statistics for
signal-level metrics. The model captures signal clustering within themes
and provides detailed posterior metrics for use in performance analysis.

## Usage

``` r
fit_bayesian_hierarchical_model(
  signal_universe_m_d_ref,
  selected_backtest_returns_corrected_positions_m_xts_upd_ref,
  selected_market_factor_proxy_m_xts_upd_ref,
  selected_backtest_returns_corrected_positions_m_upd_ref = NULL,
  elected_priors,
  selected_signal_themes_m_d_ref,
  model_spec_theme_level,
  chains,
  iter,
  warmup,
  thin,
  seed,
  adapt_delta,
  verbose,
  parallel
)
```

## Arguments

- signal_universe_m_d_ref:

  A data frame containing the signal universe. If provided, data in this
  object will be updated with posteriors.

- selected_backtest_returns_corrected_positions_m_xts_upd_ref:

  A xts containing the backtest returns data for various signals.

- selected_market_factor_proxy_m_xts_upd_ref:

  A numeric vector containing benchmark returns data. The vector will be
  recycled to match the length of the backtest returns data.

- selected_backtest_returns_corrected_positions_m_upd_ref:

  An already processed
  `selected_backtest_returns_corrected_positions_m_xts_upd_ref`. This
  data.frame is already in long format and contemplates both the
  `selected_market_factor_proxy_m_xts_upd_ref` and the
  `selected_signal_themes_m_d_ref` theme data.

- elected_priors:

  An object of class `brmsprior` containing priors for the Bayesian
  model, such as those generated by the
  `derive_informative_priors_from_data` function or provided directly by
  the user.

- selected_signal_themes_m_d_ref:

  A data frame containing metadata about signals. This data frame should
  include:

  - `tickers`: Signal identifiers matching those in
    `selected_backtest_returns_corrected_positions_m_xts_upd_ref`.

  - `theme`: Group membership for each signal, defining the clusters for
    the Bayesian hierarchical model.

  - `dates`: Dates corresponding to the backtest data. This input
    ensures proper alignment between signals and their associated
    themes.

- model_spec_theme_level:

  A character string specifying the desired Bayesian model structure.
  Options include:

  - `"random_intercept_fixed_slope"`: Includes random effects for the
    intercept at the theme level.

  - `"theme_specific_intercept_fixed_slope"`: Uses fixed intercepts for
    each theme.

  - `"theme_specific_intercept_theme_specific_slope"`: Includes fixed
    intercepts and slopes for each theme.

  - `"fixed_intercept_fixed_slope"`: Omits theme-level intercepts but
    includes random effects at the theme:signal level.

- chains:

  An integer specifying the number of Markov chains to run. Defaults to
  `4`.

- iter:

  An integer specifying the number of iterations per chain. Defaults to
  `2000`.

- warmup:

  An integer specifying the number of warmup iterations. Defaults to
  `1000`.

- thin:

  An integer specifying the thinning interval. Defaults to `1`.

- seed:

  An integer, or `NA`, specifying the random seed for reproducibility.
  No default; passed through from the caller.

- adapt_delta:

  A numeric value between 0 and 1 specifying the target acceptance rate
  for the Hamiltonian Monte Carlo sampler.

- verbose:

  A logical indicating whether to display progress messages during the
  model fitting process. Defaults to `TRUE`.

- parallel:

  A logical indicating whether to enable parallel computation using
  `future`. Defaults to `TRUE`.

## Value

A named list containing:

- `brm_model`: A `brmsfit` object containing the fitted Bayesian model.
  This object includes posterior distributions, parameter estimates,
  diagnostics, and other details of the model fit.

- `signal_universe_m_d_ref`: A data frame containing the updated
  `signal_universe_m_d_ref` with posterior summary statistics, or `NULL`
  if `signal_universe_m_d_ref` was `NULL`.

- `posterior_draws_summaries`: A list of median/89\\ (see
  [`summarize_posteriors_draws()`](https://pauloguimaraes871.github.io/factoRverse/reference/summarize_posteriors_draws.md)),
  or `NULL` if `signal_universe_m_d_ref` was `NULL`.

## Details

The function performs the following steps:

1.  **Preprocessing**: Converts the backtest returns data into long
    format, merges metadata about signals and themes, and adds the
    market factor proxy.

2.  **Formula Specification**: Defines the Bayesian model formula based
    on the chosen `model_spec_theme_level`.

3.  **Model Fitting**: Fits the Bayesian model using the `brm` function
    from the `brms` package, incorporating the provided priors and
    prepared data.

4.  **Posterior Summarization**: Extracts posterior draws and computes
    signal-level metrics such as alphas, betas, sigmas, active returns,
    tracking errors, information ratios (IR), appraisal ratios (AP), and
    Treynor ratios. Updates the `signal_universe_m_d_ref` data frame
    with these metrics.

This hierarchical approach allows the model to capture both global
(theme-level) and local (signal-level) effects while leveraging the
flexibility of Bayesian regression. The inclusion of posterior metrics
enables deeper performance analysis of signals.
